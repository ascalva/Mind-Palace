# The global event clock N: a derived causal spine — CS-a and CS-b, resolved together

> Composed at **fable** (`claude-fable-5`, 2026-07-16, the sanctioned design pass, owner-extended
> to the global clock). Filed as `draft`; ratification is owner-by-hand; `/graduate` refuses this
> note until ratified. **Design only; no build authorized.** This note is the designed **re-entry
> of two parked decisions in the ratified algebra**: CS-a ("materialized global event clock N";
> re-entry: *a consumer needs cross-stratum event-anchoring*) and CS-b ("antichain machinery for
> genuinely-partial causal order"). The consumer now exists — `dn-cross-strata-dreamer` G3 needs
> cross-stratum cuts, and φ_coh was named at the park itself — so the re-entry condition is met.
> Executing a ratified note's anticipated re-entry is not an amendment; the unpark itself is
> still owner-blessed, via ratifying THIS note.

## 1. Purpose and scope

The ratified algebra deliberately left T a **partial** meet-semilattice: cross-clock meets raise
`NoCommonClockError` "until N materializes" (`core/scope.py:311-314`), and multi-stratum point
windows raise `SliceError` without an explicit cut (`core/scope.py:441-480`). Both partialities
are honesty about one missing object — the global event clock N, the finest clock every other
clock coarsens (dn-capability-scope §2.1; Theorem S2, `cq-scope-fable-pass.md:70-76`). This note
decides what N **is**, how it is materialized without violating the architecture, the laws every
clock must satisfy against it, the cut discipline that makes cross-stratum slices lawful, what
its materialization completes (the T-meet, SLICE for non-repo strata, the dilation space, CS-f's
re-binning), and where the line between clocks and the new `Res(π)` resolution typing runs.
Out of scope: the velocity-conformal geometry and the locally-clocked superconnection (this note
supplies their *prerequisite* — a materialized N_s — not their design); any build.

## 2. Principles / decision

### 2.1 What N is: the derived causal event poset — per-store total, globally partial

**Decision GC-N1.** N is the pair `(Ev, ≼)`: the set of all **store-append events** across the
system, under the causal order generated by §2.2. It is materialized **read-side, as a derived
index over the existing stores** — never as a write-side global sequencer. Per store, `≼`
restricts to a total chain; globally it is a genuine partial order, and that partiality is a
feature (recorded concurrency), not a deficiency.

Three arguments force this shape, in descending strength:

1. **The zone-separation argument (structural — the decisive one).** A write-side global
   sequencer is a component every writer must touch at write time. The sealed core and the edge
   zone deliberately share **nothing but the async file handoff** (`core/sensing.py:9-17`
   mirrored-not-imported wire shapes; non-negotiables #1/#2: network and private data never
   share a component). A shared sequencer would be exactly such a component — a synchronous
   coupling across the one boundary the architecture refuses to couple. So a *total*,
   write-time-materialized N is not merely expensive; it is **architecturally forbidden**. What
   remains grantable is read-side derivation.
2. **The honesty argument.** The system has genuinely concurrent writers — parallel worktree
   builders, edge-zone daemons, the owner's own edits beside the scheduler — each single-writer
   *per store* (`core/stores/runledger.py:16-18`: scheduler single-writer, trough handler in
   production) but with **no global writer**. Any total order over cross-writer events would be
   an after-the-fact serialization choice; wall-clock, the obvious tiebreaker, is already ruled
   an exogenous labeling and never an event clock (ratified §2.1). The causal partial order is
   the largest structure the events actually carry.
3. **The precedent argument.** The house style is the derived consumer over existing stores with
   zero schema change (sourceset, the sweep optimizer, the σ-fibers consumer). N joins that
   family: the spine is arithmetic over what the stores already record.

**A correction to the warrant, recorded** (the σ-fibers discipline): the temporal-clocks capsule
glossed "the event-clock … is already the system's spine — the op-seq strict order." Op-seq is
the spine **of one store** (the claim-op store — a strict total order *within* it,
`supersession-lifecycle.md:33-44`); `version_seq` is per-doc, the run ledger per-run — the CS-a
park itself recorded that no single N exists. The gloss was aspirational; this note supplies the
actual object: op-seq, version chains, and run order are **per-store chains that embed into**
the derived `(Ev, ≼)`; none of them alone is N.

### 2.2 The generators of ≼ — and the audit of what the stores actually record

`≼` is the reflexive-transitive closure of three generator families. Soundness rule: **every
generator edge must witness a real recorded sequence or a real information flow. Wall-clock
timestamps are never generators** (exogenous labeling — standing rule; they may *decorate*
events for display and `Rate(wall)` denominators only).

- **g1 — per-store append chains.** Each store's own append order, where the store records one.
  The honest per-store audit (verified on disk 2026-07-16):

  | store | order recorded | chain form |
  |---|---|---|
  | versions (`core/stores/versions.py:71-81`) | `version_seq` monotonic per `doc_id` | per-doc chains (the op-seq exemplar) |
  | run ledger (`core/stores/runledger.py:130-148`) | `started_at` + insertion order; claims by `rowid` | per-store chain; run→claims is g3 |
  | edges (`core/stores/edges.py:37-44`) | `created_at` + `rowid` | chain |
  | derived/interpreted (`core/stores/derived.py:49-73`) | `created_at` | chain (ties broken by g2, never by wall) |
  | attestations (`core/attestation/store.py`) | append order + content-addressed ids | chain **plus** the g2 DAG below |
  | eval results (`eval/harness/store.py:30-44`) | **NONE — keyed only, no seq/ts column** | no chain; events enter Ev via g2/g3 alone |
  | catalog (`core/stores/catalog.py:34-35`) | keyed by path — no order | via g2/g3 |

  The eval-store row is the audit's sharpest finding: a store can be append-only *as a
  discipline* yet record no append *order*. Its events are still events; they order through
  their references (the key names the run's corpus digest and config fingerprint) — poorer
  local order, honestly represented as incomparability where no reference decides.
- **g2 — reads-from (content-address references).** An event whose row records an identifier
  minted by another event is causally after it: a run row carries `corpus_digest` +
  `config_fingerprint`; a claim carries its support digests; an eval reading carries its key
  and `evidence_ref`; a version carries its content digest; a commit carries its parents.
  **This generator is already built and mechanized in one subsystem:** the attestation
  auto-link — "any prior attestation that PRODUCED one of my inputs is a parent"
  (`core/attestation/attestor.py:59-69`, `store.producers_of`; attestation-layer §2). The spine
  *generalizes* that rule from attested actions to all store events; the attestation DAG embeds
  in `≼` as-is and is the g2 exemplar the integrity tests calibrate against.
- **g3 — recorded program order.** Where one actor's sequence is itself recorded: a run's
  `start_run → add_claim*` (`runledger.py:130-168`), a session's attestation sequence, the
  extractor's intra-dialogue op order (`supersession-lifecycle.md:33-40`).

**Soundness and incompleteness, stated as a law.** Every derived edge is true
(`≼_derived ⊆ ≼_true` — g1 by the single-writer discipline, g2 by content-address
unforgeability, g3 by the recording actor's own order). The converse fails: an unrecorded flow
(a model read a note and influenced a later artifact without a citation) leaves no edge, so
**derived concurrency over-approximates true concurrency**. Every consumer of N inherits this
caveat; §2.4's cut certification exists precisely because of it.

### 2.3 The clock laws — what every registered clock owes N

A clock is `κ = (I_κ, p_κ)` with `p_κ : Ev → I_κ` a monotone surjection (S2, established). N
materialized adds the checkable obligations:

- **Law C1 (monotone):** `e ≼ e′ ⇒ p_κ(e) ≤ p_κ(e′)`.
- **Law C2 (convex fibers):** every tick's preimage `p_κ⁻¹(i)` is order-convex — no event
  outside the tick lies ≼-between two events inside it. This is what "a commit is a **range**
  of N, not a tick" means formally (the grain caveat, temporal-clocks §1), now a property test
  rather than a slogan.
- **Law C3 (read-clocks borrow the write frontier).** `projection_event`, `last_write`, and
  `now` (`core/scope.py:171-184`) tick on *reads*, which mint no events. Their p_κ is defined
  by delegation: a projection's index is the per-store frontier it observed (the S3 remark made
  total: write-scoped clients *generate* T; read-clocks *sample* it). `MirrorView.project`'s
  tick = the vector-store chain position at projection.
- **Law C4 (wall is not a clock of Ev).** `wall` remains an exogenous coordinate, registered
  only as a `Rate(wall)` denominator (the built `telemetry_wall`,
  `eval/harness/registry.py:91-93`); it satisfies no law above and generates no order.

The registered hierarchy, now with materialized arrows: `N ⪰ N_s` (the restriction of `(Ev,≼)`
to stratum-s events — per-store chains plus their internal g2/g3 edges) `⪰ commit ⪰
distinct-snapshot`, with `commit` verified as a range (C2) against the repo. **Materializing
N_s is materializing the spine restricted to s** — which is also the standing prerequisite of
the locally-clocked superconnection and the per-stratum diachronic instruments
(temporal-clocks parked decisions; they unpark their *prerequisite* here, not their designs).

**Proper time and grain (GC-N6).** The atomic grain is one store-append event. A stratum's
proper time between two of its events is the maximal-chain length in N_s — the causal-set
construction the warrant named (external frames remain `[FROM MEMORY — verify]`: Lamport 1978;
Mattern/Fidge vector clocks; Chandy–Lamport snapshots; Bombelli–Lee–Meyer–Sorkin causal sets —
the same verification debt the ratified note already carries, discharged before any book
chapter cites them). Ordinal use (ordering) is always lawful; cardinal use (distances, Rate
denominators) must declare its grain — a commit advances N by its bundle size.

### 2.4 Cuts: certified quiescent frontiers — the SLICE rule completed

S3 established that a cross-stratum instant is a consistent cut (a down-set of `≼`; a vector
timestamp over `{N_s}`). §2.2's incompleteness means a cut computed from `≼_derived` alone may
be crossed by an *unrecorded* edge — so lawful cuts must be **certified**, not merely computed:

**Decision GC-N3.** A cut is a pair `(frontier, certificate)`: the frontier is the vector of
per-store chain positions; the certificate is one of —

1. **commit** — the repo snapshot is atomic; certifies all repo-backed strata (the ratified
   `_CUT_CLOCKS` special case, `core/scope.py:450`, now one certificate among three);
2. **trough quiescence** — the scheduler owns the job queue (`runledger.py:16-18` single-writer
   discipline); an empty queue between trough jobs certifies no in-flight core causality;
3. **handoff-empty** — the sensing handoff's `requests/` and `observations/` directories empty
   (`core/sensing.py:236-240`) certifies no in-flight edge↔core flow.

A full cross-strata cut composes certificates (commit ∧ trough-empty ∧ handoff-empty). This
types the `Scope.cut` field (today an opaque `Hashable`, `core/scope.py:469`) and **satisfies
`dn-cross-strata-dreamer` G3**: the correlator's per-stratum-anchor output discipline (its
forced v1) upgrades to lawful unified slices exactly when it reads at certified frontiers.
Chandy–Lamport marker-passing is unnecessary at this scale — quiescence is cheaper and the
scheduler already owns the fact.

**Cut falsifier:** a certified cut with a crossing g-edge (an event inside the down-set reading
from an event outside it) is a certification bug — an integrity test constructs candidate cuts
and searches for crossings; any hit fails the build.

### 2.5 What materialization completes (the ratified partialities, discharged on schedule)

- **The T-meet totalizes over registered clocks.** Theorem S2's condition ("materialized common
  refinements") becomes satisfied: `T₁ ⊓ T₂ = (N, p₁⁻¹(W₁) ∩ p₂⁻¹(W₂))` — the pullback
  intersection, a convex event set denoted by a cut-pair window `[D_lo, D_hi]` (down-set
  interval). **This inhabits CS-b's antichain machinery at the same stroke** — N-windows are
  frontier-bounded convex sets, exactly the "specified, uninhabited" grammar.
  `NoCommonClockError` narrows to unregistered/exogenous clocks (wall ⊓ anything remains an
  error); it does not vanish — the honesty survives where honesty is still due. Behavioral note:
  this executes the ratified design ("Until N exists, T is honestly a partial meet-semilattice"
  — dn-capability-scope §2.2 anticipated exactly this completion), so `TimeScope.meet` in
  `core/scope.py:334-348` changes behavior only on inputs that today raise — the old error path
  is the new computed path's falsifier surface (no previously-legal meet changes value).
- **`(N, ∗)` becomes a queryable object** — the dilation space TRA §2.3 R5 proved the ledger to
  be (Sz.-Nagy) stops being a figure of speech: "the space in which nothing was ever destroyed"
  is the spine read with the all-window.
- **φ_coh / event-anchored coherence** (CS-a's named consumer) gains its index; bp-038's
  distinct-snapshot anchoring becomes one coarsening among registered others, per its §11
  layer-on-top affordances.
- **CS-f partially unparks (GC-N7).** Rate re-binning between comparable clocks becomes
  *possible* as **re-measurement**: pull the numerator back to event level (only event-additive
  `Inv` counts qualify) and re-aggregate against κ′'s ticks — a NEW `Rate(κ′)` reading carrying
  both clocks in its provenance. Arithmetic on already-binned rates stays forbidden. CS-f's
  conservative default ("always a new measurement") is thereby kept — N merely makes the new
  measurement computable.

### 2.6 Clocks vs resolutions — the line between T and `Res(π)`, drawn once

The clock hierarchy is *fineness-ordered time*; the new `dn-resolution-result-typing` grades
values by a *resolution ruler*. They must not collapse into each other, and the test is the one
both notes already use — **capability visibility**:

- A **clock** changes which events a window denotes: `p_κ⁻¹(W)` varies with κ. T is therefore a
  scope coordinate; Rule CLOCK constrains it; transport across clocks exists (pullback along
  the hierarchy — §2.5).
- A **resolution π** changes nothing about the denoted events: every σ reads identical rows
  under an identical grant (the companion's proved invisibility). π is therefore result-side;
  Rule SCALE forbids cross-π comparison absent a declared transport.

Same medicine — carry your ruler — administered on opposite sides of the capability boundary,
and the boundary is decided by one question: *does the parameter change what the client may
see?* Clock: yes. σ/grain/depth: no. σ-persistence-along-σ (`Res(π_σ)`, no clock) and
change-along-time (`Rate(κ)`, no σ) remain orthogonal typed objects; an instrument measuring
both declares both.

### 2.7 Visibility and the firewall — N is a clock, not a data bypass

The spine stores **metadata only**: event id, store, chain position, reference ids. No payload,
no note text, no observation body — ever (falsifier: a spine row containing content is a build
bug). Even so, existence-metadata is information; the typing rule keeps the algebra consistent:

**Decision GC-N8.** An order query `order(e, e′)` or cut read is admissible under scope `s` iff
the strata of both events are within `s.Σ` — the spine answers time questions only about events
the caller could see. The spine itself is ops/derived-tier, `∉ MIRROR_READABLE`, holds no model,
opens no socket (a pure-core read-side consumer; the sealed-core import discipline unchanged).

### 2.8 Three clauses for the spine (the instrument discipline)

- *What it measures:* the derived causal order over all store-append events — per-store chains
  (g1), reads-from edges (g2), recorded program order (g3) — plus per-store frontiers and
  certified cuts.
- *Validity assumptions:* single-writer-per-store holds (the standing discipline); references
  are content-addressed hence unforgeable; `≼_derived ⊆ ≼_true` with false concurrency possible
  where flows went unrecorded (§2.2); certificates are sound (commit atomicity, queue ownership,
  handoff emptiness).
- *The observables that falsify it:* **(1)** a cycle in `≼_derived` — a forged/corrupted
  reference or generator bug; integrity test, fail-closed. **(2)** a per-store chain that fails
  to embed order-isomorphically in the spine — g1 broken. **(3)** a registered clock violating
  C1/C2 (a non-convex commit fiber = the range property broken) — property test per Clock
  member. **(4)** a certified cut with a crossing edge (§2.4). **(5)** the attestation DAG
  disagreeing with the spine's restriction to attested events — g2's exemplar is the calibration
  oracle; disagreement means the generalization, not the exemplar, is wrong.

### 2.9 Constraints honored

| constraint | binding form |
|---|---|
| Sealed core, zero egress / zones share only the handoff | read-side derivation; the sequencer alternative REJECTED structurally (§2.1-1) |
| The model advises; code acts | the spine is arithmetic over stores; no model in any path |
| Mirror firewall | metadata-only rows; Σ-gated order queries (§2.7); spine ∉ MIRROR_READABLE |
| Memory ceiling | zero models resident; DuckDB/SQLite scans at corpus scale |
| Append-only discipline | the spine derives, never writes back into any source store |
| Wall-time is not an event clock | Law C4; wall edges never generated |
| Fixed points | untouched; the frozen founding corpus is simply a flat (eventless) region of the spine |

## 3. Consequences — the build decomposition this note licenses (on ratification)

- **GC-1 — the spine skeleton** (`core/temporal/spine.py`): event enumeration over the §2.2
  audit table, g1 chains, the attestation-DAG embed, g2 reference joins, g3 program order,
  transitive reduction + the acyclicity integrity test. Read-only, model-free, no schema change
  anywhere.
- **GC-2 — the clock maps:** `p_κ` implemented per registered `Clock` member with C1/C2/C3
  property tests; commit-as-range verified against the repo; **N_s materialized as restriction**
  (the prerequisite the locally-clocked superconnection and DD-1 anchoring have been parked on).
- **GC-3 — certified cuts:** the three certificates, the composed cross-strata cut, `Scope.cut`
  typed to `(frontier, certificate)`, the crossing-edge integrity test. Unblocks
  `dn-cross-strata-dreamer` G3 (its XS-b re-entry fires when this ships *and* that note's own
  gates clear).
- **GC-4 — the T-meet completion** in `core/scope.py`: pullback meets for registered-clock
  pairs; cut-pair windows (CS-b inhabited); `NoCommonClockError` narrowed per §2.5; falsifier —
  every meet that computed *before* GC-4 returns bit-identical results after (only
  formerly-raising inputs gain values).

Order: GC-1 → GC-2 → {GC-3, GC-4}. Each is a session-sized plan; `/graduate` decides splits.
